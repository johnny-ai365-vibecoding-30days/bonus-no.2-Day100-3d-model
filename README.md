使用 ChatGPT 5.1 Thinking 和 Google AI Studio（Gemini 3 Pro Preview），兩者體驗其實都很糟糕，我上傳的是 ChatGPT 版本，但我手動處理了以下問題：
1. 函式庫 three.js 的檔案連結位置抓錯，我是自己去 google 正確的連結回來改的
2. 地圖及雲圖的材質也是抓到錯的，這邊我是去抄 Gemini 生成時用的圖資來源
3. 縮放時地球一直被裁掉，要求修改的時候一直改不好還會狡辯這是系統限制，我必需教他怎麼改

Gemini 生成的版本很棒，在 Google AI Studio 中預覽的效果很好，甚至自己串了語言模型來實現點擊後回報該地區/城市的天氣狀況的功能，但問題是我要求使用純 JavaScript 來生成時一直失敗，都會使用 TypeScript 來產生程式碼，然後要求把 TypeScript 改成 JavaScript 版本就是一連串的錯誤了，修不好，連預覽都跑不出來

但兩者相比如果要做比較正式的應用，我會選 Gemini 再想辦法部署 TypeScript ，品質真的差太多了。順便提醒，如果有要求了一些看似智能或需要調用外部資料的功能，在 Google AI Studio 中 AI 有傾向會用語言模型來實現，例如我曾做過一個黑白棋的遊戲，要求要有電腦對手陪玩，這個電腦對手在生成的程式碼就是透過串接 LLM 來實現的，會將棋譜整理成 prompt 丟給 LLM 再用回傳的結果來下棋。

看起來很棒，但如果真的部署出去，可能就會燒你的 token ，直接使用 Google AI Studio 做部署或分享的時候會有相關的警告。（講可能是因為我沒有實際部署下去，以我遇到的例子，光看程式碼內容如果部署下去最高可能性的兩個結果是：1. 部署失敗 2. 部署成功但電腦對手的功能失效。但樣本數更大的時候在 LLM 的隨機性下會不會發生什麼損及利益的意外就很難說。）
